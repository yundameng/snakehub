# SnakeHub/CowHub 应用使用文档（技术分享版）

## 主页与环境扫描介绍
你可以：
- 一眼查看本机是否已识别 Codex / Cursor / Claude
- 看到当前资源总量、重复候选和链接状态

用法：
1. 打开应用后先进入主页。
2. 点击“扫描/刷新”，确认工具识别状态。
3. 若有未识别工具，先配置对应路径再继续。

注意事项：
- 先完成一次扫描，再做导入或链接，结果更稳定。

## 首次向导介绍
你可以：
- 按“扫描 -> 导入 -> 链接”一次性完成初始接入
- 快速把历史资源纳管到中心仓库

用法：
1. 进入“首次向导”。
2. 按页面步骤完成工具扫描。
3. 选择需要导入的资源。
4. 直接在向导内执行链接。

注意事项：
- 首次接入建议只选一小批资源，验证后再批量导入。

## 本地资源导入介绍
你可以：
- 从本地目录或单文件导入 skill / hook / agent / command
- 手动创建文本资源并纳管

用法：
1. 进入“资源导入”。
2. 选择资源类型和本地路径（或手动创建）。
3. 提交导入并查看导入结果。

注意事项：
- 同类型且内容相同会自动去重，属于正常行为。

## GitLab/Git 仓库导入介绍
你可以：
- 直接填 GitLab 仓库地址导入资源
- 先浏览候选项，再精确选择需要的目录

用法：
1. 进入“仓库导入”。
2. 填写 GitLab 仓库地址（可选分支/Tag）。
3. 先点击“查看候选”。
4. 选择目标资源后执行导入。

注意事项：
- 建议“先浏览候选，再选择导入”，减少误导入。

## 路径配置（按项目定向）介绍
你可以：
- 指定某个工具类型的目标目录
- 把资源定向到某个项目目录（而不是全局目录）

用法：
1. 进入“路径配置”。
2. 选择工具（如 Codex）与资源类型（如 skill）。
3. 把路径设置为目标项目目录（如 `<项目>/.codex/skills`）。
4. 保存后回到链接页面执行链接。

注意事项：
- 推荐使用绝对路径，避免多项目切换时串目录。

## 链接管理介绍
你可以：
- 将已导入资源链接到指定工具/目录
- 自定义链接名，便于团队统一命名

用法：
1. 进入“链接管理”。
2. 选择资源、目标工具和链接名称。
3. 执行链接并记录操作编号。

注意事项：
- 链接是映射关系，不会改写资源源内容。

## 回滚与恢复介绍
你可以：
- 撤销最近一次链接操作

用法：
1. 在页面点击“回滚最近一次链接”。
2. 回滚完成后在主页或资源列表检查链接状态。

注意事项：
- 出现路径配置错误时，优先用回滚恢复到上一步。
- 当前页面仅支持“最近一次回滚”；指定 `operation_id` 回滚属于 CLI/API 能力，页面暂未提供入口。

## 典型流程：从 GitLab 到“链接到某项目”介绍
你可以：
- 用应用内流程完成“仓库导入 + 项目定向 + 链接生效”

用法：
1. 在“仓库导入”填写 GitLab 地址并导入目标 skill。
2. 在“路径配置”把 Codex skill 路径设为目标项目的 `.codex/skills`。
3. 在“链接管理”选择该 skill 执行链接。
4. 在“主页/资源列表”确认链接状态。

注意事项：
- 该流程适合多项目并行场景，每个项目分别配置路径即可。
